!split
===== Deep Learning in Context =====
Classical deep learning architectures include:

* multilayer perceptrons (MLPs),
* convolutional neural networks (CNNs),
* recurrent neural networks (RNNs).


Each architecture encodes a specific inductive bias:

* MLPs: dense global mixing,
* CNNs: locality and translation structure,
* RNNs: sequential recurrence and memory.


Transformers are also deep neural networks, but with a different structural principle: _adaptive interaction through attention._

!split
===== What Is a Transformer? =====
A transformer is a neural-network architecture built around the idea of _self-attention_.

Core principle:
  
* each input element can interact directly with every other input element,
* the interaction strengths are learned and depend on the data itself.
  
Thus transformers replace fixed interaction patterns by _data-dependent couplings_.
This makes them especially powerful for:
  
* language,
* long sequences,
* images,
* scientific fields and operator learning.

!split
===== Input as a Sequence of Vectors =====
Suppose the input is a sequence

!bt
\[
  x_1, x_2, \dots, x_n, \qquad x_i \in \mathbb{R}^d.
\]
!et
Examples:
* words in a sentence,
* patches in an image,
* nodes in a discretized field,
* spatial or temporal samples of a PDE solution.
A transformer maps this sequence to another sequence of vectors:
!bt
\[
  (x_1,\dots,x_n) \longmapsto (y_1,\dots,y_n).
\]
!et


!split
===== Self-Attention: The Basic Formula =====

For each position $i$, the transformer forms
!bt
\[
  y_i = \sum_{j=1}^n \alpha_{ij} v_j,
\]
!et
where:
* $v_j = W_V x_j$ are the _values_,
* $\alpha_{ij}$ are attention weights.
  
!split
===== How to compute weights =====
The weights are computed from
!bt
\[
  q_i = W_Q x_i, \qquad k_j = W_K x_j,
\]
!et
called the _query_ and _key_ vectors.
Then
!bt
\[
  \alpha_{ij}=\frac{\exp\!\left(q_i \cdot k_j / \sqrt{d_k}\right)}
       {\sum_{\ell=1}^n \exp\!\left(q_i \cdot k_\ell / \sqrt{d_k}\right)}.
\]
!et
This is the central mechanism of a transformer.



!split
===== Matrix Form of Attention =====


Collect the input vectors into a matrix
!bt
\[
  X \in \mathbb{R}^{n \times d}.
\]
!et
Define
!bt
\[
  Q = X W_Q,\qquad K = X W_K,\qquad V = X W_V.
\]
!et
Then the attention operation is
!bt
\[
  \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V.
\]
!et

!split
===== With equations =====
Here:
* $QK^\top$ is an $n\times n$ interaction matrix,
* the softmax normalizes each row,
* multiplication by $V$ aggregates information.
  

!split
===== Interpretation of the Attention Matrix =====
The matrix
!bt
\[
  A = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)
\]
!et
acts like a learned kernel or coupling matrix. Its entries satisfy
!bt
\[
  A_{ij} \ge 0, \qquad \sum_j A_{ij}=1.
\]
!et
Thus
!bt
\[
  y_i = \sum_j A_{ij} v_j
\]
!et
is a weighted average of the value vectors.

!split
===== Interpretation: =====
!bblock  
* query $q_i$: ``what does position \(i\) look for?''
* key $k_j$: ``what does position \(j\) offer?''
* value $v_j$: ``what information does position \(j\) contribute?''
!eblock  


!split
===== Comparison with a Standard MLP =====
A standard MLP layer has the form
!bt
\[
  y = \sigma(Wx + b),
\]
!et
with fixed weights $W$.
In contrast, attention uses
!bt
\[
  y_i = \sum_j A_{ij}(X)\, v_j,
\]
!et
where the effective coupling $A_{ij}$ depends on the input $X$.
Thus: $\text{MLP: fixed couplings} \qquad\text{vs}\qquad\text{Transformer: adaptive couplings$.
This is one reason transformers are so expressive.



!split
===== Comparison with CNNs =====

In a convolutional neural network,
!bt
\[
  y_i = \sum_{r \in \mathcal{N}(i)} w_r\, x_{i+r},
\]
!et
where $\mathcal{N}(i)$ is a small local neighborhood.
!bblock Thus CNNs assume:
* locality,
* translation invariance,
* fixed kernels/filters.
!eblock  
!bblock Attention instead uses
!bt
\[
  y_i = \sum_j A_{ij}(X)\, x_j,
\]
!et
which is
* global,
* adaptive,
* not restricted to local neighborhoods.
!eblock  



!split
===== Comparison with RNNs =====
In an RNN,
!bt
\[
  h_t = f(h_{t-1}, x_t),
\]
!et
so information propagates sequentially.
!bblock  Advantages:
* natural for time series,
* explicit recurrence.
!eblock  
!bblock  Limitations:
* long-range dependencies are hard,
* training can be unstable,
* computation is hard to parallelize.
!eblock  
Transformers avoid recurrence and instead connect all positions simultaneously through attention.



!split
===== Why the Factor $1/\sqrt{d_k}$? =====
The attention logits are
!bt
\[
 q_i \cdot k_j.
\]
!et
If $q_i$ and \(k_j\) have \(d_k\) components with comparable variance, then typically
!bt
\[
  q_i \cdot k_j \sim O(\sqrt{d_k})
\]
!et
or \(O(d_k)\) depending on scaling assumptions.
Without normalization, these logits can become large, causing the softmax to saturate. Therefore one rescales by
!bt
\[
  \frac{1}{\sqrt{d_k}}
\]
!et
to stabilize optimization and keep gradients in a useful regime.


!split ===== Multi-Head Attention}
  In practice, transformers use multiple attention heads.

  For head \(h\):
  \[
  Q^{(h)} = XW_Q^{(h)},\quad
  K^{(h)} = XW_K^{(h)},\quad
  V^{(h)} = XW_V^{(h)}.
  \]

  Each head computes
  \[
  \mathrm{head}_h
  =
  \mathrm{Attention}(Q^{(h)},K^{(h)},V^{(h)}).
  \]

  Then all heads are concatenated and linearly mixed:
  \[
  \mathrm{MultiHead}(X)
  =
  \mathrm{Concat}(\mathrm{head}_1,\dots,\mathrm{head}_H)W_O.
  \]

  Intuition:
  
  * different heads learn different interaction patterns,
  * one head may capture local structure, another long-range dependence.
  


%------------------------------------------------
!split ===== The Transformer Block}
  A standard transformer block contains:

  \begin{enumerate}
  * multi-head attention,
  * residual connection,
  * layer normalization,
  * position-wise feedforward network,
  * another residual connection and normalization.
  \end{enumerate}

  Schematically:
  \[
  X \mapsto X + \mathrm{MHA}(X)
  \]
  followed by
  \[
  X \mapsto X + \mathrm{MLP}(X).
  \]

  So a transformer is still a deep network built from familiar components:
  \[
  \text{attention} + \text{MLP} + \text{residual structure}.
  \]


%------------------------------------------------
!split ===== Positional Information}
  Attention itself is permutation-equivariant:
  \[
  (x_1,\dots,x_n)\mapsto (y_1,\dots,y_n)
  \]
  depends only on pairwise relations, not on absolute order.

  For sequence tasks we must add positional information.

  Common approaches:
  
  * learned positional embeddings,
  * sinusoidal encodings,
  * relative position encodings.
  

  For example, sinusoidal encodings use
  \[
  \mathrm{PE}(m,2r)=\sin\!\left(\frac{m}{10000^{2r/d}}\right),
  \qquad
  \mathrm{PE}(m,2r+1)=\cos\!\left(\frac{m}{10000^{2r/d}}\right).
  \]


%------------------------------------------------
!split ===== Transformers as Kernel Machines}
  Attention has the form
  \[
  y_i = \sum_j A_{ij}(X)\, v_j.
  \]

  This resembles an integral kernel operator:
  \[
  (\mathcal{K}f)(x)
  =
  \int K(x,x') f(x')\, dx'.
  \]

  In the discrete setting, attention acts like
  \[
  y_i = \sum_j K(x_i,x_j)\, v_j,
  \]
  but with a kernel \(K\) learned adaptively from the data.

  This viewpoint is useful in scientific machine learning and operator learning.


%------------------------------------------------
!split ===== Physics Viewpoint I: Adaptive Many-Body Couplings}
  A useful physics analogy is
  \[
  y_i = \sum_j J_{ij}(X)\, x_j,
  \]
  where
  \[
  J_{ij}(X) \equiv A_{ij}(X)
  \]
  is an input-dependent coupling.

  This resembles:
  
  * mean-field interaction matrices,
  * adaptive spin couplings,
  * message-passing in many-body systems,
  * self-consistent effective interactions.
  

  So transformers can be interpreted as systems with
  \[
  \text{state-dependent interaction kernels}.
  \]


%------------------------------------------------
!split ===== Physics Viewpoint II: Statistical Mechanics Analogy}
  The softmax weights are
  \[
  A_{ij} =
  \frac{e^{s_{ij}}}{\sum_\ell e^{s_{i\ell}}},
  \qquad
  s_{ij} = \frac{q_i\cdot k_j}{\sqrt{d_k}}.
  \]

  This looks like a Gibbs or Boltzmann weight:
  \[
  p_j = \frac{e^{-\beta E_j}}{Z}.
  \]

  Indeed, if we identify
  \[
  E_{ij} = - s_{ij},
  \]
  then attention weights are Gibbs-like probabilities over interaction partners.

  This suggests a statistical-mechanics interpretation:
  
  * scores \(s_{ij}\) define effective energies,
  * softmax performs a local partition-function normalization.
  


%------------------------------------------------
!split ===== Mean-Field Interpretation}
  Suppose each degree of freedom \(i\) updates by coupling to all others through effective coefficients \(A_{ij}\):
  \[
  y_i = \sum_j A_{ij} v_j.
  \]

  This resembles a mean-field update:
  \[
  m_i^{\text{new}} = F\!\left(\sum_j J_{ij} m_j\right),
  \]
  except that the couplings themselves depend on the current state.

  Thus transformers may be viewed as
  \[
  \text{nonlinear, adaptive mean-field models}.
  \]


%------------------------------------------------
!split ===== Attention and Graph Structure}
  If the input is viewed as a graph with nodes \(i\), then attention defines a complete graph with weighted edges
  \[
  i \longleftrightarrow j
  \]
  of strength \(A_{ij}\).

  This makes transformers closely related to:
  
  * graph neural networks,
  * message-passing networks,
  * nonlocal interaction models.
  

  Difference:
  
  * graph neural networks often use a fixed graph,
  * transformers learn the graph dynamically from the data.
  


%------------------------------------------------
!split ===== Transformers and PDEs: Why They Matter}
  Many PDE problems involve long-range dependencies:
  
  * elliptic equations,
  * nonlocal operators,
  * multiscale dynamics,
  * global constraints.
  

  CNNs are excellent for local structure, but transformers can naturally represent
  \[
  \text{nonlocal coupling across the whole domain}.
  \]

  This is one reason transformers have become important in scientific machine learning.



!split
===== Operator Learning Viewpoint =====
In PDEs one often wants to learn not just one solution, but an operator:
!bt
\[
  \mathcal{G}: u_0(x) \mapsto u(x,t)
\]
!et
or
!bt
\[
  a(x) \mapsto u(x),
\]
!et
where $a(x)$ might be a coefficient field.

Transformers can be used to approximate such mappings by treating the
discretized field values as tokens and learning nonlocal interactions
between them. This links transformers to:
* neural operators,
* Fourier neural operators,
* attention-based operator learning.
  

!split
===== Example: Discretized PDE fields =====
Suppose a field $u(x)$ is sampled at points
!bt
\[
  x_1,\dots,x_n.
\]
!et
Then each token may be
!bt
\[
  x_i^{\text{token}} = \big(x_i, u(x_i), a(x_i), \dots \big).
\]
!et
Attention then computes
!bt
\[
  y_i = \sum_j A_{ij} v_j,
\]
!et
which allows the representation at point \(x_i\) to depend on all other sampled points.

!split
===== Useful for =====
This is particularly useful for  
* nonlocal PDEs,
* integral equations,
* strongly coupled multiscale systems.
  


!split
===== Transformers vs Fourier Methods for PDEs =====
In scientific machine learning, Fourier-based methods and transformers both capture nonlocal structure.
!bblock Fourier-based methods:
* use spectral decomposition,
* are natural for translation-invariant problems,
* often efficient on regular grids.
!eblock  
!bblock Transformers:
* are more flexible,
* handle irregular domains and adaptive relations,
* learn data-dependent kernels rather than fixed spectral ones.
!eblock  



!split
===== Complexity of Self-Attention =====
A standard attention layer computes an $n\times n$ matrix:
!bt
\[
  QK^\top,
\]
!et  
so the cost scales as
!bt
\[
  O(n^2 d).
\]
!et
This is one of the main computational bottlenecks. By contrast:
* CNNs scale more locally,
* RNNs scale linearly in sequence length but sequentially in time.

This has motivated many sparse and efficient transformer variants.


!split
===== Why Transformers cecame so important =====
!bblock Transformers became dominant because they combine:
* global context,
* parallel computation,
* strong scaling with data and model size,
* flexible learned interaction structure.
!eblock  
They perform especially well when long-range correlations matter.
They are powerful models for systems with rich nonlocal interactions.

!split
===== Relation to Standard Deep Learning =====
Transformers are not separate from deep learning; they are a modern architecture within deep learning.

!bblock They still use:
* linear layers,
* nonlinearities,
* residual connections,
* normalization,
* gradient-based training.
!eblock  
What is new is the central primitive: $\boxed{\text{self-attention}}$ rather than convolution or recurrence.



!split
===== Strengths and Weaknesses =====

!bblock Strengths:
* captures long-range dependencies,
* highly parallelizable,
* adaptable to many data types.
!eblock  
!bblock Weaknesses:
* quadratic cost in sequence length,
* often data-hungry,
* can be less structured than domain-specific architectures.
!eblock  
In scientific applications, hybrids are often useful:
!bt
\[
  \text{science priors} + \text{attention mechanisms}.
\]
!et


!split
===== Transformers in Scientific Machine Learning =====
Active directions include:
  
* PDEs,
* turbulence modeling,
* climate and weather forecasting,
* molecular and materials modeling,
* operator learning,
* quantum many-body representations.
  
The central attraction is that attention provides a flexible way to encode interactions across scales and distances.



!split
===== Summary of this brief intro to transformers =====
Transformers are deep neural networks built around self-attention.

Mathematically:
!bt
\[
  \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V.
\]
!et

Compared with standard methods:
* MLPs use fixed dense couplings,
* CNNs use local kernels/filters,
* RNNs use sequential recurrence,
* transformers use adaptive global couplings.

This makes them especially powerful for nonlocal and multiscale problems.

A useful physical Science summary is:

* attention defines an input-dependent interaction matrix,

* transformer layers resemble adaptive many-body updates,
* this connects naturally to statistical mechanics, graph models, and operator learning.
